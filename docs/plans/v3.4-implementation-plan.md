# V3.4 Implementation Plan (Track J: Plugin Marketplace + Persistence)

> Scope: graduate the V2.2 in-process plugin runtime into a
> persisted, tenant-scoped, signed-artifact marketplace. The
> BFF owns install / upgrade / uninstall routes and file-based
> artifact storage under `apps/bff/.data/plugins/`. The
> app-shell owns the install UI and the activation envelope
> that turns an installed version into a live plugin. The
> V2.2 manifest contract is preserved; the runtime grows
> persistence types but the in-memory registry contract does
> not change. The V3.3 `Permission` union gains three new
> values (`plugin:read`, `plugin:install`, `plugin:publish`)
> gated to admins; the existing union shape stays valid for
> older consumers.
>
> This is the fifth V3 release. Track F (real auth / OIDC)
> shipped as V3.0 ([ADR 0013](../adr/0013-v3.0-closure.md)),
> Track G (production platform) shipped as V3.1
> ([ADR 0014](../adr/0014-v3.1-closure.md)), Track H
> (Tauri release pipeline) shipped as V3.2
> ([ADR 0015](../adr/0015-v3.2-closure.md)), and Track I
> (multi-tenant data model) shipped as V3.3
> ([ADR 0016](../adr/0016-v3.3-closure.md)). Track J
> depends on F (auth) for API authentication, on G
> (production platform) for the storage mount / secret
> surface, and on I (multi-tenancy) for the per-tenant
> install set; H (Tauri) is independent at the code level
> but ships before J so the operator has signed-release
> mechanics to lean on.
>
> V3.4 does **not** ship a remote registry, database-backed
> storage, GPG/Sigstore signatures, runtime signature
> verification on the in-app load path, or a marketplace
> web UI outside the app-shell -- those are V3.4.x
> follow-ups documented at the end of this plan. V3.4
> ships the **local-only, file-based, HMAC-signed,
> per-tenant** marketplace: an operator with shell access
> to the BFF can publish a plugin artifact, install it for
> a tenant, and see it activate in the app-shell; the same
> shape survives when the storage backend, the registry
> source, and the signature scheme are upgraded in V3.4.x.

## Open design questions (decided in this plan)

1. **Storage backend**: file-based under
   `apps/bff/.data/plugins/<tenantId>/<pluginId>/<version>/`
   holding `manifest.json`, `artifact.tgz`, and
   `signature.txt`. **Rationale**: V2.2 -> V3.3 had no
   plugin persistence, so the "migration" is landing the
   on-disk shape; a real database lands when the platform
   team picks one, and the `PluginStore` interface in T1 is
   the single point that needs to change. Multi-replica
   storage is a V3.4.x concern (NFS / S3 / Postgres).
2. **Artifact signing scheme**: HMAC-SHA256 with a
   per-instance secret loaded from the
   `PLUGIN_SIGNING_SECRET` env var (32-byte random in dev,
   sourced from a secret in production). **Rationale**:
   the smallest delta that proves the install-time
   verification path end-to-end. GPG / Sigstore / cosign
   are V3.4.x follow-ups when the operator wants
   third-party-verifiable provenance.
3. **Marketplace source**: local-only. The BFF accepts a
   publish POST that includes the manifest + the signed
   artifact body; there is no remote registry URL, no
   `git+https://...` clone, no npm-style index. A remote
   registry (GitHub Releases, npm, or a custom
   `@dt/plugin-index`) is a V3.4.x follow-up when the
   operator wants to ship a catalog.
4. **Versioning**: strict semver matching the V2.2
   manifest contract. Side-by-side install is allowed:
   `acme-corp` can have `hello-plugin@1.2.3` and
   `hello-plugin@1.3.0` installed simultaneously; only one
   is `active` at a time per tenant. `active` is set
   explicitly by `PUT /api/plugins/:id/activate`; a
   reinstall does not auto-activate the new version.
5. **Tenant scope**: per-tenant install set. Every plugin
   is installed for exactly one tenant; the marketplace
   route is `requiresTenantScope`-gated, and the storage
   path is keyed on `tenantId`. Sharing a plugin across
   tenants requires installing it under each tenant
   separately. A future "global plugin" feature
   (`@dt/plugin-registry` keeps a `tenantId: '*'` namespace)
   is a V3.4.x follow-up.
6. **Permission model**: the V3.3 `Permission` union gains
   three new values:
   - `plugin:read` -- list / get installed plugins
   - `plugin:install` -- install / upgrade / uninstall
     plugins for the caller's tenant
   - `plugin:publish` -- publish a new plugin artifact
     to the local registry (admin-only; covers T3 publish
     and T8 smoke admin flow)
   These are added to `ALL_PERMISSIONS` and to the `admin`
   role's `ROLE_PERMISSIONS` entry. `operator` and `viewer`
   do not gain install / publish. The V3.0 union shape
   (the 6 existing permissions) stays valid for older
   consumers; the union extension is additive and
   backward-compatible.
7. **Activation envelope**: the V2.2 in-memory
   `PluginRegistry` contract is the activation envelope.
   The BFF serves a list of installed versions to the
   app-shell, the app-shell builds a `PluginRegistration`
   per installed version, and the existing `bootstrapAppShell`
   activates them through the same registry. The runtime
   does not know about persistence, signing, or the BFF;
   it knows about manifests, registrations, and the
   state machine. This keeps the V2.2 contract stable and
   means the marketplace ships without touching
   `@dt/plugin-runtime`'s core surface (T1 only adds
   persistence types; the registry is unchanged).
8. **V2.2 hello plugin compatibility**: the existing
   `apps/web/src/plugins/hello/` plugin (shipped in V2.2)
   stays in the host's static list and continues to be
   activated through `bootstrapAppShell`. The marketplace
   path is additive: a separate `bootstrapAppShell` call
   inside `useMarketplaceInstall` reads the BFF's
   `/api/plugins/:id/installed` endpoint and appends the
   installed `PluginRegistration[]` to the static list.
   This means a V2.2 hello plugin and a V3.4 marketplace
   plugin can coexist in the same host without conflict.

  version names per `docs/development/contributing.md`
  -- "Subject rules".

  a pure-types package that depends only on
  `@dt/plugin-runtime` and `@dt/contracts`. `@dt/plugin-runtime`

## Task ordering

```
T1 -> T2 -> T3 -> T4 -> T5 -> T6 -> T7 -> T8
plugin-runtime  @dt/plugin-registry  BFF  storage  signing  tenant  UI  smoke+docs
persistence      data model           routes  on-disk  HMAC    scope  install  e2e
types                                                                      + docs
```

- T1 must land first: every T2-T7 task constructs or
  consumes an `InstallRecord` / `PluginStore` type from
  `@dt/plugin-runtime`. The persistence contract is the
  shared surface; changing it after the fact means
  rewriting the registry, the BFF routes, and the
  app-shell UI.
- T2 (`@dt/plugin-registry`) depends on T1: the
  `Plugin` / `Version` / `Install` data model imports
  `InstallRecord` from `@dt/plugin-runtime`.
- T3 (BFF marketplace routes) depends on T2: the route
  handlers consume the `PluginRegistry` / `PluginStore`
  types from `@dt/plugin-registry` and `@dt/plugin-runtime`.
- T4 (file-based storage) depends on T3: the storage
  module satisfies the `PluginStore` interface the routes
  use.
- T5 (signing) depends on T4: the storage module writes
  the signature file alongside the artifact; the install
  route verifies it before accepting the artifact.
- T6 (tenant-scope enforcement) depends on T3 + T5: the
  new `plugin:install` permission is enforced through
  the existing V3.3 `requiresTenantScope` middleware,
  and the storage path is keyed on `tenantId`.
- T7 (app-shell install UI) depends on T3 + T6: the UI
  hits the same `/api/plugins` routes the smoke hits,
  with the same `plugin:read` / `plugin:install` permission
  gating.
- T8 (smoke + docs) lands last so it captures the actual
  final interface (the route shapes, the install envelope,
  the signing key surface, the troubleshooting flow).

```ts
// packages/plugin-runtime/src/persistence.ts

/**
 * V3.4: persistence contracts for the marketplace.
 *
 * The V2.2 in-memory `PluginRegistry` is the activation
 * envelope; it does not know about persistence. This
 * module adds the types a storage backend implements
 * to describe what is installed, signed, and active for
 * a given tenant.
 *
 * Trust model: a `PluginStore` is a pure async interface.
 * The BFF wires a file-based implementation in V3.4; a
 * V3.4.x follow-up swaps it for a database. The runtime
 * never imports from a specific backend.
 */

export interface InstalledPluginVersion {
  /** Plugin id from the manifest (e.g. "hello-plugin"). */
  pluginId: string;
  /** Semver string from the manifest (e.g. "1.2.3"). */
  version: string;
  /** Tenant that owns this install. */
  tenantId: string;
  /** ISO-8601 timestamp; set when the install succeeded. */
  installedAt: string;
  /** Path to the manifest on disk (relative to the storage root). */
  manifestPath: string;
  /** Path to the signed artifact on disk (relative to the storage root). */
  artifactPath: string;
  /** Path to the signature file on disk (relative to the storage root). */
  signaturePath: string;
  /** True if this version is the active one for the tenant. */
  active: boolean;
}

export interface InstallRecord {
  tenantId: string;
  pluginId: string;
  versions: readonly InstalledPluginVersion[];
}

export interface PluginStore {
  /** List every installed version for a tenant, across all plugins. */
  listForTenant(tenantId: string): Promise<readonly InstallRecord[]>;
  /** List every installed version for a single plugin in a tenant. */
  listVersions(tenantId: string, pluginId: string): Promise<readonly InstalledPluginVersion[]>;
  /** Get one installed version, or null if not installed. */
  getVersion(
    tenantId: string,
    pluginId: string,
    version: string,
  ): Promise<InstalledPluginVersion | null>;
  /** Get the active version for a tenant + plugin, or null if none active. */
  getActive(tenantId: string, pluginId: string): Promise<InstalledPluginVersion | null>;
  /** Persist a new install. Returns the recorded `InstalledPluginVersion`. */
  putVersion(record: InstalledPluginVersion): Promise<InstalledPluginVersion>;
  /** Mark `version` as the active one; previous active (if any) is cleared. */
  setActive(
    tenantId: string,
    pluginId: string,
    version: string,
  ): Promise<InstalledPluginVersion>;
  /** Remove an installed version. Returns true if a row was deleted. */
  removeVersion(
    tenantId: string,
    pluginId: string,
    version: string,
  ): Promise<boolean>;
}

/**
 * Compare two semver strings. Returns negative if a<b,
 * zero if equal, positive if a>b. Pre-release tags are
 * ordered lower than the same MAJOR.MINOR.PATCH without
 * a pre-release tag (per semver 2.0.0).
 *
 * Exported so the BFF can reject a downgrade in T5
 * (installing 1.2.3 over an active 1.3.0 returns 409).
 */
export function compareSemver(a: string, b: string): number {
  const pa = parseSemver(a);
  const pb = parseSemver(b);
  if (!pa || !pb) {
    throw new Error(`invalid semver: ${!pa ? a : b}`);
  }
  if (pa.major !== pb.major) return pa.major - pb.major;
  if (pa.minor !== pb.minor) return pa.minor - pb.minor;
  if (pa.patch !== pb.patch) return pa.patch - pb.patch;
  // No pre-release tag > pre-release tag with the same numbers.
  if (pa.preRelease && !pb.preRelease) return -1;
  if (!pa.preRelease && pb.preRelease) return 1;
  if (pa.preRelease && pb.preRelease) {
    return pa.preRelease < pb.preRelease
      ? -1
      : pa.preRelease > pb.preRelease
        ? 1
        : 0;
  }
  return 0;
}

interface ParsedSemver {
  major: number;
  minor: number;
  patch: number;
  preRelease: string | null;
}

function parseSemver(s: string): ParsedSemver | null {
  const m = /^(\d+)\.(\d+)\.(\d+)(?:-([0-9A-Za-z-.]+))?(?:\+[0-9A-Za-z-.]+)?$/.exec(s);
  if (!m) return null;
  return {
    major: Number(m[1]),
    minor: Number(m[2]),
    patch: Number(m[3]),
    preRelease: m[4] ?? null,
  };
}
```

**Step 1.1: Interface is async-only**

`PluginStore` is fully async. The V3.4 file-based
implementation in T4 is async; a V3.4.x database
implementation will also be async. The runtime does not
ship a synchronous fallback because that would force the
file backend to read everything upfront.

**Step 1.2: ASCII / format check**

`pluginId`, `tenantId`, `compareSemver`, the new types
are all ASCII. The semver parser uses the same regex
shape the V2.2 manifest validator uses, so the test
suite for the manifest validator's semver check is
already green for the same inputs.

**Step 1.3: Test coverage**

`persistence.test.ts` covers:

- `compareSemver('1.2.3', '1.2.3')` returns 0
- `compareSemver('1.2.3', '1.2.4')` returns negative
- `compareSemver('2.0.0', '1.99.99')` returns positive
- `compareSemver('1.0.0', '1.0.0-beta')` returns positive
  (no pre-release > pre-release)
- `compareSemver('1.0.0-beta.2', '1.0.0-beta.1')` returns
  positive (pre-release identifiers compared
  lexicographically per semver 2.0.0)
- `compareSemver('1.0.0+build', '1.0.0')` returns 0
  (build metadata ignored)
- `compareSemver('not-semver', '1.0.0')` throws
- The `PluginStore` interface is exported and has all
  7 methods (interface contract smoke test; an in-memory
  fake `PluginStore` is implemented inline so the test
  imports compile without needing T4's file backend).

```ts
// packages/plugin-registry/src/registry.ts

/**
 * V3.4: the marketplace data model.
 *
 * `Plugin` describes the canonical plugin (id + name +
 * vendor); `PluginVersion` describes a single published
 * version (the manifest + signature + artifact path).
 * The in-memory `createInMemoryPluginIndex` factory is
 * a test-friendly implementation of `RegistryIndex`; the
 * BFF wires a file-based one in T4.
 *
 * Boundary: this package depends on `@dt/plugin-runtime`
 * (for `InstallRecord` / `InstalledPluginVersion`) and
 * `@dt/contracts` (for `PluginManifest`). It does not
 * import `vue`, `three`, `pinia`, or any BFF module.
 */

import type {
  InstallRecord,
  InstalledPluginVersion,
} from '@dt/plugin-runtime';
import type { PluginManifest } from '@dt/contracts';

export interface PluginVersion {
  /** Plugin id from the manifest. */
  pluginId: string;
  /** Semver version. */
  version: string;
  /** Manifest for this version; parsed and validated. */
  manifest: PluginManifest;
  /** Path to the signed artifact on disk (relative to the storage root). */
  artifactPath: string;
  /** Path to the signature file on disk (relative to the storage root). */
  signaturePath: string;
  /** ISO-8601 timestamp; set when the artifact was published. */
  publishedAt: string;
}

export interface Plugin {
  /** Plugin id; the canonical key for `PluginStore`. */
  id: string;
  /** Human-readable name (from the manifest). */
  name: string;
  /** Vendor (from the manifest). */
  vendor: string;
  /** Versions of this plugin available in the local registry, ordered by semver. */
  versions: readonly PluginVersion[];
}

export interface RegistryIndex {
  /** All plugins in the local registry, keyed by id. */
  list(): Promise<readonly Plugin[]>;
  /** Get one plugin (all versions); null if not in the registry. */
  get(pluginId: string): Promise<Plugin | null>;
  /** Publish a new plugin version to the local registry. */
  publish(version: PluginVersion): Promise<PluginVersion>;
}

/**
 * Pure in-memory registry for tests. Production uses the
 * file-based implementation in apps/bff/src/plugins/.
 */
export function createInMemoryPluginIndex(): RegistryIndex {
  const plugins = new Map<string, Plugin>();
  return {
    async list() {
      return [...plugins.values()].sort((a, b) => a.id.localeCompare(b.id));
    },
    async get(pluginId) {
      return plugins.get(pluginId) ?? null;
    },
    async publish(version) {
      const existing = plugins.get(version.pluginId);
      const next: Plugin = {
        id: version.pluginId,
        name: version.manifest.name,
        vendor: version.manifest.vendor,
        versions: existing
          ? [...existing.versions, version].sort((a, b) =>
              a.version.localeCompare(b.version),
            )
          : [version],
      };
      plugins.set(version.pluginId, next);
      return version;
    },
  };
}
```

The BFF route module:

```ts
// apps/bff/src/routes/marketplace.ts (V3.4)

import { Hono } from 'hono';
import { requiresTenantScope } from '../middleware/requires-tenant.js';
import type { PluginStore } from '@dt/plugin-runtime';
import type { RegistryIndex } from '@dt/plugin-registry';
import {
  writePluginArtifact,
  verifyArtifactSignature,
} from '../plugins/storage.js';
import type {
  PublishPluginRequest,
  InstallPluginRequest,
  ActivatePluginRequest,
} from '@dt/contracts';

export function marketplaceRoutes(
  store: PluginStore,
  index: RegistryIndex,
): Hono {
  const app = new Hono();

  // List the local registry (all plugins, all versions).
  // `plugin:read` is enough; viewers get this too.
  app.get('/plugins', requiresTenantScope(store as never, 'plugin:read'), async (c) => {
    const list = await index.list();
    return c.json(list);
  });

  // Publish a new plugin version. Admin-only via
  // `plugin:publish`; the route is not tenant-scoped at
  // the route layer because publishing is a global
  // (registry-wide) action. The tenant scope still
  // requires a session; the publisher is recorded but
  // not associated with a tenant install.
  app.post(
    '/plugins',
    requiresTenantScope(store as never, 'plugin:publish'),
    async (c) => {
      const body = (await c.req.json()) as PublishPluginRequest;
      const written = await writePluginArtifact(body);
      const published = await index.publish({
        pluginId: body.manifest.id,
        version: body.manifest.version,
        manifest: body.manifest,
        artifactPath: written.artifactPath,
        signaturePath: written.signaturePath,
        publishedAt: new Date().toISOString(),
      });
      return c.json(published, 201);
    },
  );

  // Get one plugin (all versions).
  app.get('/plugins/:id', requiresTenantScope(store as never, 'plugin:read'), async (c) => {
    const id = c.req.param('id');
    const plugin = await index.get(id);
    if (!plugin) {
      return c.json({ error: 'PLUGIN_NOT_FOUND', message: id }, 404);
    }
    return c.json(plugin);
  });

  // Install a plugin version for the caller's tenant.
  // `plugin:install`; the tenant is read from
  // `c.var.tenant` set by `requiresTenantScope`.
  app.post(
    '/plugins/:id/install',
    requiresTenantScope(store as never, 'plugin:read'),
    async (c) => {
      const { tenant } = c.var;
      const id = c.req.param('id');
      const body = (await c.req.json()) as InstallPluginRequest;
      const plugin = await index.get(id);
      if (!plugin) {
        return c.json({ error: 'PLUGIN_NOT_FOUND', message: id }, 404);
      }
      const version = plugin.versions.find((v) => v.version === body.version);
      if (!version) {
        return c.json(
          { error: 'PLUGIN_VERSION_NOT_FOUND', message: body.version },
          404,
        );
      }
      // Verify signature before accepting the install.
      const valid = await verifyArtifactSignature(
        version.artifactPath,
        version.signaturePath,
      );
      if (!valid) {
        return c.json({ error: 'PLUGIN_SIGNATURE_INVALID', message: id }, 400);
      }
      const existing = await store.listVersions(tenant.tenant.id, id);
      if (existing.some((v) => v.version === body.version)) {
        return c.json(
          {
            error: 'PLUGIN_ALREADY_INSTALLED',
            message: `${id}@${body.version}`,
          },
          409,
        );
      }
      const record = await store.putVersion({
        pluginId: id,
        version: body.version,
        tenantId: tenant.tenant.id,
        installedAt: new Date().toISOString(),
        manifestPath: version.artifactPath.replace(/artifact\.tgz$/, 'manifest.json'),
        artifactPath: version.artifactPath,
        signaturePath: version.signaturePath,
        active: existing.length === 0,
      });
      return c.json(record, 201);
    },
  );

  // List installed versions for the caller's tenant.
  app.get(
    '/plugins/:id/installed',
    requiresTenantScope(store as never, 'plugin:read'),
    async (c) => {
      const { tenant } = c.var;
      const id = c.req.param('id');
      const versions = await store.listVersions(tenant.tenant.id, id);
      return c.json(versions);
    },
  );

  // Activate an installed version.
  app.put(
    '/plugins/:id/activate',
    requiresTenantScope(store as never, 'plugin:read'),
    async (c) => {
      const { tenant } = c.var;
      const id = c.req.param('id');
      const body = (await c.req.json()) as ActivatePluginRequest;
      const versions = await store.listVersions(tenant.tenant.id, id);
      if (!versions.some((v) => v.version === body.version)) {
        return c.json(
          { error: 'PLUGIN_VERSION_NOT_FOUND', message: body.version },
          404,
        );
      }
      const activated = await store.setActive(tenant.tenant.id, id, body.version);
      return c.json(activated);
    },
  );

  // Uninstall an installed version.
  app.delete(
    '/plugins/:id/installed/:version',
    requiresTenantScope(store as never, 'plugin:read'),
    async (c) => {
      const { tenant } = c.var;
      const id = c.req.param('id');
      const version = c.req.param('version');
      const removed = await store.removeVersion(tenant.tenant.id, id, version);
      if (!removed) {
        return c.json(
          { error: 'PLUGIN_VERSION_NOT_FOUND', message: version },
          404,
        );
      }
      return c.json({ ok: true });
    },
  );

  return app;
}
```

```ts
// apps/bff/src/plugins/storage.ts (V3.4)

import { promises as fs } from 'node:fs';
import * as path from 'node:path';
import type {
  InstallRecord,
  InstalledPluginVersion,
  PluginStore,
} from '@dt/plugin-runtime';
import type { PublishPluginRequest } from '@dt/contracts';

const STORAGE_ROOT = path.resolve(
  process.env.PLUGIN_STORAGE_ROOT ?? '.data/plugins',
);

/**
 * V3.4 file-based plugin storage.
 *
 * Layout on disk:
 *
 *   <root>/<tenantId>/<pluginId>/<version>/manifest.json
 *   <root>/<tenantId>/<pluginId>/<version>/artifact.tgz
 *   <root>/<tenantId>/<pluginId>/<version>/signature.txt
 *   <root>/<tenantId>/<pluginId>/<version>/install.json
 *   <root>/<tenantId>/<pluginId>/active.txt   (semver of the active version)
 *   <root>/_registry/<pluginId>/<version>/...  (publish-time artifacts)
 *
 * The publish-time files are keyed under `_registry/`
 * (no tenant); the install-time files are keyed under
 * `<tenantId>/`. The two trees never overlap; the
 * `_` prefix keeps `listForTenant` from accidentally
 * scanning the registry tree.
 *
 * V3.4 ships JSON files because the volume is small
 * (a few hundred plugins at most in V3.4); a V3.4.x
 * follow-up swaps this for SQLite or Postgres when the
 * volume or the multi-replica story needs it.
 */

interface ActivePointer {
  active: string;
}

interface InstallMeta {
  pluginId: string;
  version: string;
  tenantId: string;
  installedAt: string;
  manifestPath: string;
  artifactPath: string;
  signaturePath: string;
}

export class FilePluginStore implements PluginStore {
  constructor(private readonly root: string = STORAGE_ROOT) {}

  private tenantDir(tenantId: string, pluginId: string, version: string): string {
    return path.join(this.root, tenantId, pluginId, version);
  }

  private activePointerPath(tenantId: string, pluginId: string): string {
    return path.join(this.root, tenantId, pluginId, 'active.txt');
  }

  async listForTenant(tenantId: string): Promise<readonly InstallRecord[]> {
    const tenantDir = path.join(this.root, tenantId);
    let entries: string[] = [];
    try {
      entries = await fs.readdir(tenantDir);
    } catch (err) {
      if ((err as NodeJS.ErrnoException).code === 'ENOENT') return [];
      throw err;
    }
    const records: InstallRecord[] = [];
    for (const pluginId of entries) {
      const versions = await this.listVersions(tenantId, pluginId);
      if (versions.length > 0) {
        records.push({ tenantId, pluginId, versions });
      }
    }
    return records;
  }

  async listVersions(
    tenantId: string,
    pluginId: string,
  ): Promise<readonly InstalledPluginVersion[]> {
    const pluginDir = path.join(this.root, tenantId, pluginId);
    let entries: string[] = [];
    try {
      entries = await fs.readdir(pluginDir);
    } catch (err) {
      if ((err as NodeJS.ErrnoException).code === 'ENOENT') return [];
      throw err;
    }
    const active = await this.readActivePointer(tenantId, pluginId);
    const versions: InstalledPluginVersion[] = [];
    for (const version of entries) {
      if (version === 'active.txt') continue;
      const versionDir = path.join(pluginDir, version);
      const stat = await fs.stat(versionDir);
      if (!stat.isDirectory()) continue;
      const manifestPath = path.join(versionDir, 'manifest.json');
      const artifactPath = path.join(versionDir, 'artifact.tgz');
      const signaturePath = path.join(versionDir, 'signature.txt');
      const manifestStat = await fs.stat(manifestPath);
      versions.push({
        pluginId,
        version,
        tenantId,
        installedAt: manifestStat.mtime.toISOString(),
        manifestPath,
        artifactPath,
        signaturePath,
        active: active === version,
      });
    }
    return versions.sort((a, b) => a.version.localeCompare(b.version));
  }

  async getVersion(
    tenantId: string,
    pluginId: string,
    version: string,
  ): Promise<InstalledPluginVersion | null> {
    const versions = await this.listVersions(tenantId, pluginId);
    return versions.find((v) => v.version === version) ?? null;
  }

  async getActive(
    tenantId: string,
    pluginId: string,
  ): Promise<InstalledPluginVersion | null> {
    const active = await this.readActivePointer(tenantId, pluginId);
    if (!active) return null;
    return this.getVersion(tenantId, pluginId, active);
  }

  async putVersion(record: InstalledPluginVersion): Promise<InstalledPluginVersion> {
    const dir = this.tenantDir(record.tenantId, record.pluginId, record.version);
    await fs.mkdir(dir, { recursive: true });
    const meta: InstallMeta = {
      pluginId: record.pluginId,
      version: record.version,
      tenantId: record.tenantId,
      installedAt: record.installedAt,
      manifestPath: record.manifestPath,
      artifactPath: record.artifactPath,
      signaturePath: record.signaturePath,
    };
    await fs.writeFile(path.join(dir, 'install.json'), JSON.stringify(meta, null, 2));
    if (record.active) {
      await this.writeActivePointer(record.tenantId, record.pluginId, record.version);
    }
    return record;
  }

  async setActive(
    tenantId: string,
    pluginId: string,
    version: string,
  ): Promise<InstalledPluginVersion> {
    const existing = await this.getVersion(tenantId, pluginId, version);
    if (!existing) {
      throw new Error(`plugin not installed: ${pluginId}@${version}`);
    }
    await this.writeActivePointer(tenantId, pluginId, version);
    return { ...existing, active: true };
  }

  async removeVersion(
    tenantId: string,
    pluginId: string,
    version: string,
  ): Promise<boolean> {
    const dir = this.tenantDir(tenantId, pluginId, version);
    try {
      await fs.rm(dir, { recursive: true, force: true });
    } catch (err) {
      if ((err as NodeJS.ErrnoException).code === 'ENOENT') return false;
      throw err;
    }
    const active = await this.readActivePointer(tenantId, pluginId);
    if (active === version) {
      const pointerPath = this.activePointerPath(tenantId, pluginId);
      try {
        await fs.rm(pointerPath, { force: true });
      } catch {
        // ignore
      }
    }
    return true;
  }

  private async readActivePointer(tenantId: string, pluginId: string): Promise<string | null> {
    const pointerPath = this.activePointerPath(tenantId, pluginId);
    try {
      const buf = await fs.readFile(pointerPath, 'utf8');
      const parsed = JSON.parse(buf) as ActivePointer;
      return parsed.active;
    } catch (err) {
      if ((err as NodeJS.ErrnoException).code === 'ENOENT') return null;
      throw err;
    }
  }

  private async writeActivePointer(
    tenantId: string,
    pluginId: string,
    version: string,
  ): Promise<void> {
    const pointerPath = this.activePointerPath(tenantId, pluginId);
    await fs.mkdir(path.dirname(pointerPath), { recursive: true });
    await fs.writeFile(pointerPath, JSON.stringify({ active: version }));
  }
}
```

```ts
// apps/bff/src/plugins/signing.ts (V3.4)

import { createHmac, randomBytes, timingSafeEqual } from 'node:crypto';

/**
 * V3.4: HMAC-SHA256 plugin artifact signing.
 *
 * The signing secret is loaded from `PLUGIN_SIGNING_SECRET`
 * (32-byte random in dev; from a Kubernetes Secret /
 * Hashicorp Vault / etc. in production). The BFF refuses
 * to boot if the env var is missing or shorter than 32
 * bytes -- the marketplace path is unreachable without
 * a secret.
 *
 * The signature is a base64-encoded HMAC-SHA256 of the
 * artifact body, written to `signature.txt` next to the
 * artifact. Verification recomputes the HMAC and compares
 * with `crypto.timingSafeEqual` to avoid timing leaks.
 *
 * V3.4.x follow-ups: GPG / Sigstore / cosign for
 * third-party-verifiable provenance.
 */

let SECRET: Buffer | null = null;

export function loadSigningSecret(): Buffer {
  if (SECRET) return SECRET;
  const raw = process.env.PLUGIN_SIGNING_SECRET;
  if (!raw || raw.length < 32) {
    throw new Error(
      'PLUGIN_SIGNING_SECRET must be set to a 32-byte (or longer) string',
    );
  }
  SECRET = Buffer.from(raw, 'utf8');
  return SECRET;
}

export function signArtifact(artifact: Buffer): string {
  const hmac = createHmac('sha256', loadSigningSecret());
  hmac.update(artifact);
  return hmac.digest('base64');
}

export function verifySignature(
  artifact: Buffer,
  signatureB64: string,
): boolean {
  const expected = Buffer.from(signArtifact(artifact), 'base64');
  const provided = Buffer.from(signatureB64, 'base64');
  if (expected.length !== provided.length) return false;
  return timingSafeEqual(expected, provided);
}

export function generateDevSigningSecret(): string {
  return randomBytes(32).toString('base64');
}
```

```ts
// apps/bff/src/plugins/storage.ts (T5 implementation)

import { promises as fs } from 'node:fs';
import * as path from 'node:path';
import { signArtifact } from './signing.js';
import type { PublishPluginRequest } from '@dt/contracts';

const STORAGE_ROOT = path.resolve(
  process.env.PLUGIN_STORAGE_ROOT ?? '.data/plugins',
);

export async function writePluginArtifact(
  body: PublishPluginRequest,
): Promise<{ artifactPath: string; signaturePath: string }> {
  const artifact = Buffer.from(body.artifact, 'base64');
  const signature = signArtifact(artifact);
  const dir = path.join(
    STORAGE_ROOT,
    '_registry',
    body.manifest.id,
    body.manifest.version,
  );
  await fs.mkdir(dir, { recursive: true });
  const artifactPath = path.join(dir, 'artifact.tgz');
  const signaturePath = path.join(dir, 'signature.txt');
  const manifestPath = path.join(dir, 'manifest.json');
  await Promise.all([
    fs.writeFile(artifactPath, artifact),
    fs.writeFile(signaturePath, signature),
    fs.writeFile(manifestPath, JSON.stringify(body.manifest, null, 2)),
  ]);
  return { artifactPath, signaturePath };
}

export async function verifyArtifactSignature(
  artifactPath: string,
  signaturePath: string,
): Promise<boolean> {
  const [artifact, signature] = await Promise.all([
    fs.readFile(artifactPath),
    fs.readFile(signaturePath, 'utf8'),
  ]);
  return verifySignature(artifact, signature.trim());
}
```

```ts
// apps/bff/src/plugins/policy.ts (V3.4)

import type { Permission } from '@dt/contracts';
import type { AuthStore } from '../auth/store.js';

/**
 * V3.4: marketplace policy gate.
 *
 * `requiresTenantScope` (V3.3) already enforces the
 * tenant boundary (the session's tenant claim must
 * resolve through the tenant registry). This module
 * adds the marketplace-specific rule that operators /
 * viewers cannot install / publish even though they
 * have `plugin:read`. The union extension in T3 grants
 * `plugin:read` to operators and viewers; this policy
 * is what keeps install / publish behind admin.
 */

export async function canInstallForTenant(
  store: AuthStore,
  headers: Headers,
  tenantId: string,
): Promise<{ allowed: boolean; reason?: string }> {
  const me = await store.getMe(headers);
  if (!me.session) return { allowed: false, reason: 'no session' };
  if (me.session.tenantId !== tenantId) {
    return { allowed: false, reason: 'tenant mismatch' };
  }
  const permissions = me.session.permissions ?? [];
  if (!permissions.includes('plugin:install' as Permission)) {
    return { allowed: false, reason: 'missing plugin:install' };
  }
  return { allowed: true };
}

export async function canPublish(
  store: AuthStore,
  headers: Headers,
): Promise<{ allowed: boolean; reason?: string }> {
  const me = await store.getMe(headers);
  if (!me.session) return { allowed: false, reason: 'no session' };
  const permissions = me.session.permissions ?? [];
  if (!permissions.includes('plugin:publish' as Permission)) {
    return { allowed: false, reason: 'missing plugin:publish' };
  }
  return { allowed: true };
}
```

pass through; the explicit `canInstallForTenant` check
is what enforces `plugin:install`. This split keeps
the existing V3.3 middleware untouched and adds the
marketplace policy as a separate concern.

**Step 6.1: Cross-tenant install attempt**

The V3.3 `requiresTenantScope` already returns 403 if
the session's `tenantId` does not match the URL's
tenant. The marketplace has no URL tenant (the install
is keyed on `c.var.tenant`); the `canInstallForTenant`
helper is what enforces the boundary. A session for
`acme-corp` cannot install for `globex-ind` because the
helper's `me.session.tenantId !== tenantId` check
returns false. The cross-tenant case is therefore
collapsed: the install always targets the caller's
tenant, and the `canInstallForTenant` helper is the
single enforcement point.

```ts
// packages/app-shell/src/composables/useMarketplaceInstall.ts (V3.4)

import { ref, computed, type Ref } from 'vue';
import type {
  InstallRecord,
  InstalledPluginVersion,
  PluginRegistration,
} from '@dt/plugin-runtime';
import { validatePluginManifest } from '@dt/plugin-runtime';
import { usePluginStore } from '../stores/plugin.js';

/**
 * V3.4 marketplace install composable.
 *
 * Bridges the BFF's `/api/plugins/:id/installed`
 * endpoint to the V2.2 plugin runtime: each installed
 * version becomes a `PluginRegistration` that the
 * runtime's `createPluginRegistry` activates. The
 * runtime stays host-agnostic; this composable is the
 * host-side glue.
 *
 * The composable does NOT do dynamic JS imports. It
 * reads the manifest JSON from the BFF and constructs
 * a stub `activate` that, for V3.4, renders the
 * manifest's name + version into a placeholder panel.
 * V3.4.x adds real dynamic loading (URL-based
 * `import()` behind a CSP and signature re-check).
 */

export interface MarketplaceApi {
  listInstalled(tenantId: string): Promise<readonly InstallRecord[]>;
  install(
    tenantId: string,
    pluginId: string,
    version: string,
  ): Promise<InstalledPluginVersion>;
  activate(
    tenantId: string,
    pluginId: string,
    version: string,
  ): Promise<InstalledPluginVersion>;
  uninstall(tenantId: string, pluginId: string, version: string): Promise<void>;
  /** Read the manifest for an installed version (V3.4 stub returns null). */
  readManifest(
    tenantId: string,
    pluginId: string,
    version: string,
  ): Promise<unknown>;
}

export function useMarketplaceInstall(api: MarketplaceApi, tenantId: Ref<string>) {
  const installed = ref<readonly InstallRecord[]>([]);
  const loading = ref(false);
  const error = ref<string | null>(null);

  const pluginStore = usePluginStore();

  async function refresh(): Promise<void> {
    loading.value = true;
    error.value = null;
    try {
      installed.value = await api.listInstalled(tenantId.value);
      pluginStore.replaceMarketplaceRegistrations(
        await buildRegistrations(installed.value, api, tenantId.value),
      );
    } catch (err) {
      error.value = err instanceof Error ? err.message : String(err);
    } finally {
      loading.value = false;
    }
  }

  async function install(pluginId: string, version: string): Promise<void> {
    await api.install(tenantId.value, pluginId, version);
    await refresh();
  }

  async function activate(pluginId: string, version: string): Promise<void> {
    await api.activate(tenantId.value, pluginId, version);
    await refresh();
  }

  async function uninstall(pluginId: string, version: string): Promise<void> {
    await api.uninstall(tenantId.value, pluginId, version);
    await refresh();
  }

  const activeByPlugin = computed(() => {
    const map = new Map<string, string>();
    for (const record of installed.value) {
      const active = record.versions.find((v) => v.active);
      if (active) map.set(record.pluginId, active.version);
    }
    return map;
  });

  return {
    installed,
    loading,
    error,
    activeByPlugin,
    refresh,
    install,
    activate,
    uninstall,
  };
}

async function buildRegistrations(
  records: readonly InstallRecord[],
  api: MarketplaceApi,
  tenantId: string,
): Promise<readonly PluginRegistration[]> {
  const out: PluginRegistration[] = [];
  for (const record of records) {
    for (const v of record.versions) {
      if (!v.active) continue;
      const manifest = await api.readManifest(tenantId, record.pluginId, v.version);
      if (!manifest) continue;
      const ok = validatePluginManifest(manifest);
      if (!ok.ok) continue;
      out.push({
        manifest: ok.manifest,
        activate: () => [
          {
            kind: 'ui-panel',
            panel: {
              id: `${ok.manifest.id}-${ok.manifest.version}`,
              title: `${ok.manifest.name} (${ok.manifest.version})`,
              component: {
                render() {
                  return `${ok.manifest.name} v${ok.manifest.version}`;
                },
              },
            },
          },
        ],
      });
    }
  }
  return out;
}
```

## Known caveats (deliberate, recorded for the closure ADR)

- **File-based storage in production code path**. V3.4
  uses `FilePluginStore` under `apps/bff/.data/plugins/`.
  A real database (SQLite for single-replica, Postgres
  for multi-replica) is a V3.4.x follow-up; the
  `PluginStore` interface in `@dt/plugin-runtime` is the
  single point that needs to change. Multi-replica
  deployment with the file backend is unsupported
  because each replica has its own filesystem.
- **HMAC-SHA256 per-instance secret**. V3.4 signs with
  `PLUGIN_SIGNING_SECRET` (per-instance). The signature
  is not third-party-verifiable; an attacker with the
  secret can sign malicious artifacts. GPG / Sigstore /
  cosign are V3.4.x follow-ups when the operator wants
  third-party-verifiable provenance.
- **Local-only marketplace source**. V3.4 publishes via
  `POST /api/plugins`; there is no remote registry, no
  git clone, no npm-style index. A remote registry
  (GitHub Releases, npm, or a custom
  `@dt/plugin-index`) is a V3.4.x follow-up when the
  operator wants to ship a catalog.
- **No runtime signature verification on the in-app load
  path**. V3.4 verifies signatures at install time
  (server-side); the app-shell reads the manifest from
  the install record and renders a placeholder panel
  without re-verifying the signature. Real dynamic
  loading (URL-based `import()` behind CSP) with a
  signature re-check is a V3.4.x follow-up. The V2.2
  trust model (same-origin static import, contract-
  based) still applies to marketplace plugins; the
  marketplace is the install envelope, not the load
  envelope.
- **Stub activation envelope**. V3.4's
  `useMarketplaceInstall` composable renders
  marketplace-installed plugins as placeholder panels
  (manifest name + version). Real activation (the
  plugin's `activate` function running its real
  logic) is a V3.4.x follow-up; V3.4 proves the
  install + activate plumbing without committing to a
  dynamic-import shape.
- **Single-replica fan-out for publish / install**. The
  V3.4 file backend assumes a single BFF replica;
  multi-replica needs shared storage (NFS, S3, or a
  database). V3.4.x concern.
- **Permission union grew**. V3.4 adds `plugin:read`,
  `plugin:install`, `plugin:publish`. The V3.0 stable
  contract is preserved (additive extension); consumers
  that exhaustively switch on the union need a
  compile-time bump to add the new cases. The V3.4
  test suite covers this.
- **release-please artifact caveat**: per ADR 0014
  deviations, the release-please run after V3.4 may
